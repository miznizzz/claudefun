# Creative Constraint Generator

Constraints are how creativity gets unstuck. The brain hates open space — it
freezes. Give it a wall and it finds a door. This skill generates the right
kind of constraint for whatever you're stuck on.

---

## Creative Domains & Example Constraints

| Domain | Example Constraints |
|---|---|
| Writing (essays, newsletters, blogs) | POV shifts, word limits, structure inversions, forbidden words |
| Journaling / personal writing | Time limits, sensory anchors, single-object focus, emotional polarity |
| Visual / design work | Palette restrictions, one element only, asymmetry rule |
| Cooking | Single hero ingredient, one pan only, cuisine mash-up, "use what's left" |
| Aesthetic / spatial decisions | One colour only, no buying allowed, repurpose something existing |
| Any creative task | Time box (20 min), opposite day, beginner's mind, steal like an artist |

---

## Constraint Types

**Structural constraints** — change the *shape* of the thing
- "Write it in second person"
- "No paragraphs longer than 2 sentences"
- "Start in the middle"
- "End before the conclusion"

**Material constraints** — limit what you're allowed to use
- "One main ingredient / colour / element"
- "Nothing new — only what you already have"
- "50 words maximum"

**POV constraints** — shift who's speaking or seeing
- "Write it as if you're explaining to a sceptic"
- "What would the antagonist say?"
- "Zoom in to one specific moment instead of the whole thing"

**Time constraints** — create urgency
- "You have 15 minutes. Go."
- "Write the worst version first, as fast as possible"
- "First draft in one sitting, no editing"

**Inversion constraints** — flip the premise
- "What's the opposite of what you were going to say?"
- "What if the problem is actually a feature?"
- "Start with the ending"

---

## Output Format

Give the user **one constraint** — not a menu. If they don't like it, generate another.

Format:
> **[Constraint name]:** [The constraint in one punchy sentence]
> *Why this works for what you're stuck on: [one line explanation]*
> Optional: a micro-example of what it might look like applied

---

## Hard Rules

- ❌ Never give a list of constraints — pick one and commit to it
- ❌ Don't explain creativity theory — get to the constraint fast
- ✅ The constraint should feel slightly uncomfortable — that's the point
- ✅ If they hate it, generate a completely different type (not a variation of the same one)
- ✅ Match the energy of what they're stuck on — don't give a playful constraint to something emotionally heavy
